reset三个参数的对比

soft: 仅仅在本地库移动HEAD指针

mixed: 在本地哭移动HEAD指针，重置缓存区。

hard: 在本地库移动HEAD指针，重置缓存区，重置工作区。

git与远程库的交互

在github创建远程库

  • 点击New Repository,根据指示进行操作即可

将本地库添加到远程库

  • 找到github远程库的地址，并且复制
  • 在本地项目目录下执行以下命令 origin是给推送的仓库起的别名
  • 将本地库首次推送到远程库

协同合作

  • 克隆远程库到本地
  • 仓库创建者添加合作者的github账号到Collabrators，按照github网站提示进行操作即可
  • 合作开发者在github上接受邀请

解决协同合作的冲突

  • 修改冲突文件，手动解冲突
  • 执行添加暂存区的命令git add <file>
  • 执行提交的命令git commit -m "comments"，注意，此时的提交不能带文件名

协同合作的另一种形式

  • 协同开发者在github上fork项目仓库到自己的远程库
  • 开发者在自己本地clone仓库，此时的url为自己账号下的远程仓库url
  • 本地操作，然后推送到远程
  • pull request

  • 执行命令
  • 进入.ssh目录
  • 将id_rsa.pub文件里面的内容复制添加到github下的SSH keys中
